# ProxysVPN Desktop

> 🇬🇧 [English version](README.md)

Нативный VPN-клиент для macOS под сервис [ProxysVPN](https://proxysvpn.com). Пропускает **весь** системный трафик (браузер, Telegram, Discord, игры, App Store) через VLESS Reality + ML-KEM туннель — никаких ручных настроек proxy в системе не нужно.

## Возможности

- **VPN на всю систему** через TUN-устройство (`utun225`) и `tun2socks` — каждое TCP/UDP соединение в системе идёт через туннель
- **VLESS Reality** с post-quantum **ML-KEM** обменом ключами — устойчив к DPI, неотличим от обычного HTTPS
- **One-click установщик** в DMG — копирует, запускает. Никакого терминала
- **Один запрос пароля** при запуске VPN (нужен для установки сетевых маршрутов)
- **Системный трей** с Показать / Отключить / Выход; закрытие окна не разрывает VPN
- **Живой пинг** через туннель, реконнект при смене сети
- **Встроенный просмотр логов** — кнопка в нижнем правом углу для диагностики
- **Светлая / тёмная тема** следует системной macOS

## Установка

1. Скачай **`ProxysVPN_0.1.0_aarch64.dmg`** из [последнего релиза](https://github.com/evilork/proxysvpn-desktop/releases/latest)
2. Открой `.dmg` (двойной клик)
3. Двойной клик на **«ProxysVPN Installer»**
4. Нажми **«Установить»** → введи пароль администратора если попросит
5. Готово — приложение запустится автоматически

> При первом запуске Installer macOS может спросить «открыть приложение от неустановленного разработчика» — нажми **«Открыть»** или зайди в *Системные настройки → Приватность и безопасность → Открыть всё равно*. Это разовое действие.

После запуска введи свою ссылку подписки (получить можно на [proxysvpn.com](https://proxysvpn.com)) и нажми кнопку питания.

## Системные требования

- macOS 11 Big Sur или новее
- Apple Silicon (M1/M2/M3/M4). Поддержка Intel Mac запланирована на v0.2.

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│  ProxysVPN.app                                                  │
│                                                                 │
│  ┌─────────────────┐    запускает  ┌──────────────────────────┐ │
│  │ React UI        │ ────────────> │ xray (VLESS Reality)     │ │
│  │ (Tauri 2)       │               │ + ML-KEM post-quantum    │ │
│  └─────────────────┘               └──────────────────────────┘ │
│           │                                  ▲                  │
│           │                                  │ SOCKS5 :10808    │
│           ▼                                  │                  │
│  ┌─────────────────┐                ┌──────────────────────────┐│
│  │ Rust бэкенд     │ ─ запускает ─> │ tun2socks → utun225      ││
│  │ (lib.rs)        │                │ + split-default routes   ││
│  └─────────────────┘                └──────────────────────────┘│
│                                              │                  │
└──────────────────────────────────────────────┼──────────────────┘
                                               ▼
                                       Весь системный трафик
                                       (Telegram, игры, etc.)
```

- **Фронтенд:** React 19 + TypeScript + Tailwind CSS + neumorphic дизайн
- **Бэкенд:** Rust (Tauri 2) + tokio
- **Встроенные бинари:** xray-core, tun2socks (скачиваются `scripts/fetch-binaries.sh` при сборке)
- **Reality crypto:** ML-KEM-768 hybrid x25519 (post-quantum)

## Сборка из исходников

```bash
# Требуется: Rust 1.80+, Node.js 18+, npm
git clone https://github.com/evilork/proxysvpn-desktop.git
cd proxysvpn-desktop

# Скачать xray + tun2socks (~25 МБ)
bash src-tauri/scripts/fetch-binaries.sh

# Установить зависимости JS
npm install

# Собрать релизный .app + .dmg с встроенным Installer
bash src-tauri/scripts/build-release.sh
bash src-tauri/scripts/bundle-installer-into-dmg.sh
```

Подписанный `.dmg` появится в `src-tauri/target/release/bundle/dmg/`.

## Приватность

- **Никакой телеметрии.** Приложение общается только с твоей ссылкой подписки и с VLESS-сервером который она вернёт.
- **Логи остаются локально.** `~/Library/Logs/ProxysVPN/app.log` никуда не уходит — поделись им с саппортом вручную если нужно.
- **Никаких аккаунтов в десктоп-клиенте.** Авторизация через ссылку подписки (длинный неугадываемый токен привязанный к твоему профилю на ProxysVPN).

## Решение проблем

| Проблема | Что делать |
|---|---|
| Installer показывает «от неустановленного разработчика» | *Системные настройки → Приватность и безопасность → Открыть всё равно* |
| Приложение зависает на запуске / нет запроса пароля | Полностью закрой, затем `sudo pkill -9 -x proxysvpn-desktop`, перезапусти |
| Подключилось, но Telegram/Instagram не грузят | Проблема с DPI-фрагментацией. Открой логи (нижний правый угол) и пришли экспорт в саппорт |
| VPN отваливается каждые несколько минут | Кликни на иконку (i) в углу → Скачать .txt → пришли в саппорт |
