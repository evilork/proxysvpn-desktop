# Установка ProxysVPN на macOS

## Системные требования

- macOS 11 (Big Sur) или новее
- Apple Silicon (M1/M2/M3/M4) — поддерживается
- Intel Mac — пока не поддерживается

## Шаг 1. Установка

1. Скачайте `ProxysVPN_X.X.X_aarch64.dmg` с сайта
2. Откройте `.dmg` файл
3. Перетащите `ProxysVPN.app` в папку `Applications`
4. Закройте окно установщика и извлеките `ProxysVPN` (правый клик на иконке диска → "Извлечь")

## Шаг 2. Разрешение запуска (один раз)

macOS блокирует запуск приложений от незарегистрированных разработчиков. ProxysVPN распространяется без подписи Apple Developer ID — это никак не влияет на безопасность, но требует одного дополнительного шага.

Выберите **один** из вариантов:

### Вариант A. Через Terminal (быстрый)

Откройте Terminal (Cmd + Space → "Terminal") и выполните:

```bash
xattr -cr /Applications/ProxysVPN.app
```

После этого ProxysVPN запускается обычным двойным кликом.

### Вариант B. Через Finder (без Terminal)

1. Откройте папку `Applications` в Finder
2. **Правой кнопкой мыши** на `ProxysVPN.app` → **Открыть**
3. В появившемся окне нажмите **Открыть**
4. Введите пароль администратора (нужен для VPN-туннеля)

В дальнейшем приложение запускается обычным двойным кликом.

## Что делать если…

**"Приложение повреждено и не может быть открыто"**
Выполните в Terminal:
```bash
xattr -cr /Applications/ProxysVPN.app
```

**"Запрещено системой безопасности"**
System Settings → Privacy & Security → пролистайте вниз → нажмите **"Open Anyway"** напротив ProxysVPN.

**VPN не подключается после установки**
Перезагрузите Mac — это нужно для регистрации сетевого расширения. Если не помогло — Terminal:
```bash
sudo rm -f /tmp/proxysvpn.pid
```
Затем перезапустите ProxysVPN.

**Не запрашивает пароль администратора**
Удалите `.app` и установите заново. ProxysVPN всегда запрашивает пароль при первом запуске после установки.

## Удаление

```bash
sudo rm -rf /Applications/ProxysVPN.app
sudo rm -f /tmp/proxysvpn.pid /var/run/proxysvpn.pid 2>/dev/null
```

## Безопасность

ProxysVPN подписан **локально** (ad-hoc подпись). Это значит:
- Подпись подтверждает целостность файла после сборки разработчиком
- Подпись **не** подтверждает личность Apple — поэтому Gatekeeper блокирует запуск без явного разрешения пользователя
- Когда мы оформим Apple Developer ID, шаг 2 уйдёт автоматически

Сверяйте SHA256 скачанного файла с опубликованным на сайте перед установкой:

```bash
shasum -a 256 ~/Downloads/ProxysVPN_*.dmg
```
